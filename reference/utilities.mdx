description: "API reference for TuLIP utilities: cooler and sillburp_cooler execute 2D slice simulations; cubemaker builds and saves 3D sill geometry archives."
---

The `utilities` module provides three standalone functions that sit one level above `sill_controls` in the TuLIP workflow. `cooler` and `sillburp_cooler` each execute a full 2-D cross-section simulation (including heat diffusion and carbon-emission modelling) by reading pre-built sill-cube slices from disk. `cubemaker` generates and saves one 3-D sill cube for a single `(flux, volume)` combination. All three functions are designed to be called in parallel across many parameter combinations using a task scheduler such as `joblib`.

---

## `cooler`

```python
cooler(
    iter, z_index, flux,
    lat_range=None, sc=None,
    diff_val=31.536, temp_grad_base=30/1000,
    file_path_dir='sillcubes/',
    post_cooling_time=30000,
    saving_factor=[100],
    x=None, y=None, dx=None, dy=None,
    rock_prop_dict=None,
    save_dir_footnote=None
)
```

Runs a complete SILLi-based 2-D slice simulation. The function reads the pre-computed initial thermogenic state from VTK files, loads the appropriate sill-cube slice, constructs a time array that spans from the thermal-maturation end time to the last sill emplacement plus `post_cooling_time`, and calls `sill_controls.emplace_sills` with `model='silli'`. Output is written as a CSV file containing the time series of CO₂, melt areas, and solidus areas at the specified `save_dir`.

### Parameters

<ParamField path="iter" type="int" required>
  Row index into the `n_sills.csv` file, selecting which volume iteration to run.
</ParamField>

<ParamField path="z_index" type="int" required>
  Z-slice index extracted from the 3-D sill cube. Identifies which lateral cross-section this simulation represents.
</ParamField>

<ParamField path="flux" type="float" required>
  Magma flux (m³ yr⁻¹). Used to locate the correct subdirectory under `file_path_dir`.
</ParamField>

<ParamField path="lat_range" type="list | None" default="None">
  Lateral-range identifier used as a subdirectory suffix when sill cubes were built with a specific lateral range. Pass `None` to use the flux directory directly.
</ParamField>

<ParamField path="sc" type="sill_controls | None" default="None">
  Pre-constructed `sill_controls` instance. When `None`, one is instantiated from the remaining grid parameters using default settings.
</ParamField>

<ParamField path="diff_val" type="float" default="31.536">
  Typical thermal diffusivity (m² yr⁻¹) used to compute the stable time step `dt = (min(dx, dy))² / (5 × diff_val)`.
</ParamField>

<ParamField path="temp_grad_base" type="float" default="0.03">
  Geothermal gradient at the base of the modelled section (°C m⁻¹), used to compute the Neumann bottom-boundary heat flux `q = k[-1, :] × temp_grad_base`.
</ParamField>

<ParamField path="file_path_dir" type="str" default="'sillcubes/'">
  Root directory containing all pre-computed inputs: VTK property/carbon state files, sill-cube numpy arrays, and emplacement parameter CSVs.
</ParamField>

<ParamField path="post_cooling_time" type="float" default="30000">
  Additional simulation time after the last sill emplacement (years) to capture post-intrusion cooling.
</ParamField>

<ParamField path="saving_factor" type="list[int]" default="[100]">
  VTK output cadence passed to `emplace_sills`. Every `saving_factor[0]`-th time step is saved.
</ParamField>

<ParamField path="x" type="float | None" required>
  Horizontal extent of the model domain (m). Must be provided.
</ParamField>

<ParamField path="y" type="float | None" required>
  Vertical extent of the model domain (m). Must be provided.
</ParamField>

<ParamField path="dx" type="float | None" required>
  Lateral grid spacing (m). Must be provided.
</ParamField>

<ParamField path="dy" type="float | None" required>
  Vertical grid spacing (m). Must be provided.
</ParamField>

<ParamField path="rock_prop_dict" type="dict | None" default="None">
  Custom rock-property dictionary. Falls back to `sc.rock_prop_dict` when `None`.
</ParamField>

<ParamField path="save_dir_footnote" type="str | None" default="None">
  Optional suffix appended to the auto-constructed save directory path, useful for labelling sensitivity runs.
</ParamField>

### Returns

This function does not return a value. It writes two output files to the run directory:

- `times.csv` — columns: `time_steps`, `tot_RCO2`, `melt10`, `melt50`, `area_sills`, `tot_solidus`.
- VTK property snapshots at the cadence set by `saving_factor` (written by `emplace_sills`).

### Example

```python
from TuLIP import sill_controls
import utilities as util

sc = sill_controls(x=300_000, y=12_000, dx=50, dy=50)

util.cooler(
    iter=0,
    z_index=5,
    flux=30e9,
    sc=sc,
    x=300_000, y=12_000, dx=50, dy=50,
    file_path_dir='sillcubes/',
    post_cooling_time=30_000,
    saving_factor=[100],
)
```

To parallelise across slices and volume iterations:

```python
from joblib import Parallel, delayed

Parallel(n_jobs=8)(
    delayed(util.cooler)(
        iter=i, z_index=z, flux=30e9,
        sc=sc, x=300_000, y=12_000, dx=50, dy=50,
    )
    for i in range(5) for z in range(10)
)
```

## `sillburp_cooler`

```python
sillburp_cooler(
    iter, z_index, flux,
    lat_range=None, sc=None,
    diff_val=31.536, temp_grad_base=30/1000,
    file_path_dir='sillcubes/',
    post_cooling_time=30000,
    saving_factor=[100],
    x=None, y=None, dx=None, dy=None,
    rock_prop_dict=None,
    save_dir_footnote=None
)
```

Identical in structure to `cooler` but uses the Sillburp multi-kerogen model (`model='sillburp'`) instead of SILLi. It reads additional pre-computed state files — `progress_of_reactions.npy`, `rate_of_reactions.npy`, `oil.npy` (or `oil.csv`), and `sillburp_weights.csv` — before calling `emplace_sills`.

### Parameters

All parameters are identical to `cooler`. Consult that section for full descriptions. The key differences in the source files read are:

- Reads `initial_sillburp_state_properties.vtk` and `initial_sillburp_state_carbon.vtk` instead of SILLi equivalents.
- Loads `progress_of_reactions.npy` and `rate_of_reactions.npy` for the multi-kerogen state.
- Loads `oil.npy` (or `oil.csv`) for the oil-generation accumulator.
- Optionally loads `sillburp_weights.csv` for spatially varying kerogen-channel weights.

<ParamField path="iter" type="int" required>
  Volume-iteration index.
</ParamField>

<ParamField path="z_index" type="int" required>
  Z-slice index.
</ParamField>

<ParamField path="flux" type="float" required>
  Magma flux (m³ yr⁻¹).
</ParamField>

<ParamField path="lat_range" type="list | None" default="None">
  Lateral-range subdirectory identifier.
</ParamField>

<ParamField path="sc" type="sill_controls | None" default="None">
  Pre-constructed `sill_controls` instance.
</ParamField>

<ParamField path="diff_val" type="float" default="31.536">
  Reference diffusivity for stable `dt` calculation.
</ParamField>

<ParamField path="temp_grad_base" type="float" default="0.03">
  Basal geothermal gradient (°C m⁻¹).
</ParamField>

<ParamField path="file_path_dir" type="str" default="'sillcubes/'">
  Root directory for all pre-computed inputs.
</ParamField>

<ParamField path="post_cooling_time" type="float" default="30000">
  Post-emplacement cooling duration (years).
</ParamField>

<ParamField path="saving_factor" type="list[int]" default="[100]">
  VTK output cadence.
</ParamField>

<ParamField path="x, y, dx, dy" type="float" required>
  Grid dimensions and spacing (m).
</ParamField>

<ParamField path="rock_prop_dict" type="dict | None" default="None">
  Custom rock-property dictionary.
</ParamField>

<ParamField path="save_dir_footnote" type="str | None" default="None">
  Optional suffix for the save directory.
</ParamField>

### Returns

No return value. Writes `times.csv` (columns: `time_steps`, `tot_RCO2`, `melt10`, `melt50`, `area_sills`, `tot_solidus`) and VTK snapshots to the run directory.

### Example

```python
import utilities as util
from TuLIP import sill_controls

sc = sill_controls(x=300_000, y=12_000, dx=50, dy=50)

util.sillburp_cooler(
    iter=0,
    z_index=3,
    flux=30e9,
    sc=sc,
    x=300_000, y=12_000, dx=50, dy=50,
    file_path_dir='sillcubes/',
    post_cooling_time=50_000,
)
```

## `cubemaker`

```python
cubemaker(
    tot_volume, flux,
    x, y, z, dx, dy,
    maturation_time, save_dir,
    sc=None,
    lat_range=None,
    thickness_range=None,
    aspect_ratio=None,
    depth_range=None,
    shape=None,
    depth_function=None,
    lat_function=None,
    dims_function=None,
    emplace_dike=False,
    orientations=None,
    k=31.536
)
```

Builds a single 3-D sill cube for the specified `(tot_volume, flux)` pair and saves it in multiple formats (`.npy` for fast loading, `.vtk` for visualisation, `.csv` for emplacement parameters). Also validates the cube by checking that `n_sills` from `build_sillcube` matches the length of the emplacement parameter array, retrying up to five times if the check fails.

Phase times are constructed internally: thermal maturation runs for `maturation_time`, the emplacement phase runs for `tot_volume / flux`, and a cooling phase of 10⁶ / dt × dt years follows.

### Parameters

<ParamField path="tot_volume" type="float" required>
  Target total magma volume (m³).
</ParamField>

<ParamField path="flux" type="float" required>
  Magma supply rate (m³ yr⁻¹). Used both as a simulation parameter and as the name of the output subdirectory.
</ParamField>

<ParamField path="x" type="float" required>
  Horizontal extent of the model domain (m).
</ParamField>

<ParamField path="y" type="float" required>
  Vertical extent of the model domain (m).
</ParamField>

<ParamField path="z" type="float" required>
  Z-dimension extent (m). Sets the number of 2-D slices in the cube.
</ParamField>

<ParamField path="dx" type="float" required>
  Lateral grid spacing (m).
</ParamField>

<ParamField path="dy" type="float" required>
  Vertical grid spacing (m).
</ParamField>

<ParamField path="maturation_time" type="float" required>
  Pre-emplacement thermal maturation duration (years). Rounded to the nearest multiple of `dt`.
</ParamField>

<ParamField path="save_dir" type="str" required>
  Root output directory. A subdirectory named after `flux` is created automatically.
</ParamField>

<ParamField path="sc" type="sill_controls | None" default="None">
  Pre-constructed `sill_controls` instance. When `None`, one is created with the supplied grid dimensions and default settings.
</ParamField>

<ParamField path="lat_range" type="list[float] | None" default="None">
  `[x_min, x_max, sd_x]` for sill lateral positions. Defaults to `[x/3, 2x/3, x/6]`.
</ParamField>

<ParamField path="thickness_range" type="list[float] | None" default="None">
  `[min, max, sd]` for sill thickness. Defaults to `[100, 600, 500]` m.
</ParamField>

<ParamField path="aspect_ratio" type="list[float] | None" default="None">
  `[mean, sd]` for width/thickness ratio. Defaults to `[19.23, 9.74]`.
</ParamField>

<ParamField path="depth_range" type="list[float] | None" default="None">
  `[min, max, sd]` for emplacement depth. Defaults to `[500, 4000, 5000]` m.
</ParamField>

<ParamField path="shape" type="str | None" default="None">
  `'elli'` (default) or `'rect'`.
</ParamField>

<ParamField path="depth_function" type="str | callable | None" default="None">
  Depth-sampling distribution: `'normal'` (default), `'uniform'`, or `'empirical'`.
</ParamField>

<ParamField path="lat_function" type="str | callable | None" default="None">
  Lateral-position distribution: `'normal'` (default), `'uniform'`, or `'empirical'`.
</ParamField>

<ParamField path="dims_function" type="str | callable | None" default="None">
  Dimension-sampling distribution: `'normal'` (default), `'uniform'`, `'scaled'`, or `'empirical'`.
</ParamField>

<ParamField path="emplace_dike" type="bool" default="False">
  Include vertical feeder dike tails.
</ParamField>

<ParamField path="orientations" type="numpy.ndarray | None" default="None">
  Dike orientation angles (degrees). Randomly sampled when `emplace_dike=True` and `None`.
</ParamField>

<ParamField path="k" type="float" default="31.536">
  Reference diffusivity (m² yr⁻¹) for stable `dt` calculation.
</ParamField>

### Returns

<ResponseField name="n_sills_array" type="list[int]">
  Single-element list containing the actual number of sills built into the cube after volume truncation.
</ResponseField>

### Files written

Under `save_dir/{flux:.3e}/`:

| File | Description |
|---|---|
| `sillcube{tot_volume}.npy` | Raw string array, fastest to load |
| `sillcube{tot_volume}.vtk` | Integer-encoded cube for ParaView |
| `emplacement_params{tot_volume}.csv` | Per-sill parameters: `empl_times`, `empl_heights`, `x_space`, `width`, `thickness` |

### Example

```python
import utilities as util
from TuLIP import sill_controls

sc = sill_controls(x=300_000, y=12_000, dx=50, dy=50)

n_sills = util.cubemaker(
    tot_volume=0.5e6 * 1e9,
    flux=30e9,
    x=300_000, y=12_000, z=300_000,
    dx=50, dy=50,
    maturation_time=1e6,
    save_dir='sillcubes',
    sc=sc,
    lat_range=[100_000, 200_000, 50_000],
    thickness_range=[100, 600, 500],
    aspect_ratio=[19.23, 9.74],
    depth_range=[500, 4000, 5000],
    shape='elli',
)
print(f"Built {n_sills[0]} sills")
```

To build cubes for multiple flux values in parallel:

```python
from joblib import Parallel, delayed

fluxes = [10e9, 20e9, 30e9]
Parallel(n_jobs=3)(
    delayed(util.cubemaker)(
        tot_volume=0.5e6 * 1e9,
        flux=f,
        x=300_000, y=12_000, z=300_000,
        dx=50, dy=50,
        maturation_time=1e6,
        save_dir='sillcubes',
        sc=sc,
    )
    for f in fluxes
)
```
