The model domain in TuLIP is a 2D rectangular grid representing a vertical cross-section through the crust. Each cell in the grid holds a stack of physical properties — temperature, lithology, porosity, density, TOC, and specific heat — that evolve as sills are emplaced and heat diffuses through the rock. Before you can run any simulation, you must tell TuLIP how wide and how deep your crustal block is, and how finely to resolve it.

## Grid parameters

The four required arguments to `sill_controls` define the physical size and resolution of your domain.

<ParamField path="x" type="float" required>
  Horizontal extent of the crustal block in meters. This sets how wide the 2D cross-section is. For Large Igneous Province (LIP) simulations, values on the order of hundreds of kilometres are typical (e.g., `300000` for 300 km).
</ParamField>

<ParamField path="y" type="float" required>
  Vertical extent (depth) of the crustal block in meters. This controls how much of the crust — from surface to the base of the lithosphere or mantle — is included in the model. A value of `12000` (12 km) covers a typical upper-crustal sedimentary sequence; deeper mantle lithosphere simulations may use `60000` or more.
</ParamField>

<ParamField path="dx" type="float" required>
  Grid spacing in the horizontal direction in meters. Together with `x`, this determines the number of columns in the temperature and property arrays: `b = int(x // dx)`. Finer spacing increases spatial accuracy but multiplies memory use and solver time.
</ParamField>

<ParamField path="dy" type="float" required>
  Grid spacing in the vertical direction in meters. Together with `y`, this determines the number of rows in the arrays: `a = int(y // dy)`. Vertical resolution also controls how thin a sill can be faithfully represented — a sill thinner than `dy` will be under-resolved.
</ParamField>

## Choosing grid resolution

Grid resolution is the primary lever controlling the accuracy–speed tradeoff in TuLIP. Finer grids resolve thin sills and sharp thermal gradients more accurately, but they increase the array size quadratically and tighten the stability constraint on explicit solvers (see [Solvers](/configuration/solvers)).

For most LIP simulations you should set `dx` and `dy` equal to each other to produce isotropic cells. As a starting point:

- **50–100 m** — high resolution; appropriate for detailed studies of individual sill aureoles or when you need accurate TOC profiles near contacts.
- **100–250 m** — balanced resolution; recommended for basin-scale or multi-sill ensemble runs where many realisations are needed.
- **>250 m** — coarse; useful for rapid sensitivity tests or when `x` and `y` are very large, but sill thickness and aureole width will be poorly resolved.

<Tip>
  Start with `dx=dy=250` to prototype your setup quickly, then refine to `dx=dy=50` for production runs. A factor-of-five reduction in spacing increases the grid point count by 25× and cuts your explicit solver's stable time step by the same factor.
</Tip>

## Domain size guidance

LIP cross-sections typically span tens to hundreds of kilometres horizontally and tens of kilometres vertically. A common configuration for an upper-crustal sedimentary basin study is:

| Dimension | Typical value | Notes |
|-----------|--------------|-------|
| `x` | 100 000–500 000 m | Covers the lateral extent of a volcanic province |
| `y` | 10 000–60 000 m | Depth of the crust to the Moho or deeper |

<Note>
  The `x` dimension also controls how many independent 2D slices you can extract from a pre-computed 3D sill cube when you call `generate_sill_2D_slices()`. The number of available slices equals `int(x // dx)`, so a wider domain with finer `dx` gives you more statistically independent realisations from a single stochastic sill geometry.
</Note>

## Temperature initialization

TuLIP expects the temperature field to be stored as the first layer of the `props_array` (index `0`). A realistic starting condition is a depth-dependent geothermal gradient. Use NumPy's `linspace` to build a 1D gradient and then broadcast it across the horizontal axis:

```python
import numpy as np
from TuLIP import sill_controls

# Define domain
sc = sill_controls(x=300000, y=12000, dx=50, dy=50)

a = int(sc.y // sc.dy)  # number of rows
b = int(sc.x // sc.dx)  # number of columns

# Create a geothermal gradient: 20 °C at surface to 400 °C at base
T_surface = 20.0   # °C
T_base    = 400.0  # °C
T_column  = np.linspace(T_surface, T_base, a)

# Broadcast to a full 2D temperature field
T_field = np.tile(T_column[:, np.newaxis], (1, b))
```

You can also incorporate a Moho temperature step, radiogenic heat production offsets, or any other geothermal profile — just populate `T_field` before assembling `props_array`.

## Complete instantiation example

The minimal call to `sill_controls` requires only the four grid parameters:

```python
from TuLIP import sill_controls

sc = sill_controls(
    x=300000,
    y=12000,
    dx=50,
    dy=50
)
```

This creates an object with a 240 × 6000 cell grid (12 km deep, 300 km wide at 50 m resolution). All other parameters take their defaults: constant thermal conductivity (`kc_val=7.884e7`), built-in rock property table, and basalt melt as the intrusive lithology. See [Rock Properties](/configuration/rock-properties) to override any of these defaults.
